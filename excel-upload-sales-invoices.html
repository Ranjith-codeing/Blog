Excel Upload for Sales Invoices in Business Central

Designing a robust, scalable bulk-upload pipeline with row-level validation, structured error reporting, and background reprocessing — without touching the BC database until every row is clean.

Ranjith · March 2026

Bulk invoice entry is one of the most common pain points in any BC implementation. Finance teams wrestle with Excel files every month, manually keying hundreds of rows into the system. This guide builds the entire upload pipeline end-to-end — from file parsing to posting — with every failure mode handled.

The Problem with Manual Entry

When a client sends 300 invoices at month-end, the standard Business Central UI handles them one at a time. A data entry operator might take two full days to key all of them — with inevitable typos, wrong G/L accounts, and missing dimensions. Overtime costs mount. Errors require credit notes. Auditors find inconsistencies.

A well-built upload tool collapses that two-day effort into twenty minutes: the operator drops an Excel file, reviews the validation report, fixes any issues, and hits Post. That's the goal this guide delivers.

💡
Design principle: Never write a single BC record until all rows have passed validation. Partial imports create reconciliation nightmares.

Excel Template Design

The template is the contract between the finance team and the system. Keep it opinionated: named columns, locked headers, dropdown validation for key fields. When users deviate from the template, parsing fails fast with a clear message.

#Column NameTypeMandatoryNotes
ACustomer No.Code[20]✅Must exist in BC Customer table
BPosting DateDate✅Format: DD/MM/YYYY
CDocument DateDate✅Format: DD/MM/YYYY
DExternal Doc. No.Text[35]✅Unique per customer
EItem No.Code[20]✅Must exist in BC Item table
FDescriptionText[100]❌Defaults to Item Description if blank
GQuantityDecimal✅Must be > 0
HUnit PriceDecimal✅Excl. VAT
ILine Discount %Decimal❌0–100
JDimension 1 CodeCode[20]❌Department code
KDimension 2 CodeCode[20]❌Project code
⚠️
Column order is fixed. Do not add or remove columns from the template. The parser reads by position, not by header name, to avoid issues with localised column names.

Staging Table Approach

Never parse directly into Sales Header and Sales Line. Use a staging table as an intermediate buffer. This decouples parsing from validation, enables row-level error tracking, and allows reprocessing of failed rows without re-uploading the entire file.

Table: Sales Invoice Upload Buffer

table 50200 "Sales Invoice Upload Buffer"
{
    DataClassification = CustomerContent;

    fields
    {
        field(1;  "Entry No.";          Integer)        { AutoIncrement = true; }
        field(2;  "Upload Batch No.";   Code[20])       { }
        field(3;  "Row No.";            Integer)        { }
        field(4;  "Customer No.";       Code[20])       { }
        field(5;  "Posting Date";       Date)           { }
        field(6;  "Document Date";      Date)           { }
        field(7;  "External Doc. No.";  Text[35])       { }
        field(8;  "Item No.";           Code[20])       { }
        field(9;  "Description";        Text[100])      { }
        field(10; "Quantity";           Decimal)        { }
        field(11; "Unit Price";         Decimal)        { }
        field(12; "Line Discount %";    Decimal)        { }
        field(13; "Dimension 1 Code";   Code[20])       { }
        field(14; "Dimension 2 Code";   Code[20])       { }
        field(15; Status;               Enum "Upload Row Status") { }
        field(16; "Error Message";      Text[2048])     { }
        field(17; "Created At";         DateTime)       { }
        field(18; "Processed At";       DateTime)       { }
        field(19; "BC Invoice No.";     Code[20])       { }
    }

    keys
    {
        key(PK; "Entry No.") { Clustered = true; }
        key(Batch; "Upload Batch No.", Status) { }
        key(Row; "Upload Batch No.", "Row No.") { }
    }
}

AL Code: Parsing the File

Business Central's ExcelBuffer table handles reading .xlsx files without any external libraries. The parser reads the sheet row by row, maps each cell to a staging buffer record, and assigns a batch number for tracking. 

codeunit 50200 "Sales Invoice Upload Parser"
{
    procedure ParseAndStage(FileInStream: InStream; var BatchNo: Code[20]): Integer
    var
        ExcelBuf: Record "Excel Buffer" temporary;
        Buffer:   Record "Sales Invoice Upload Buffer";
        RowCount: Integer;
        LastRow:  Integer;
        RowNo:    Integer;
    begin
        // Generate a unique batch identifier
        BatchNo := CopyStr(Format(CreateGuid()), 1, 20);

        // Load the Excel file into the temporary buffer
        ExcelBuf.LockTable();
        ExcelBuf.OpenBookStream(FileInStream, 'Invoices'); // sheet name
        ExcelBuf.ReadSheet();

        // Find the last row with data
        ExcelBuf.SetFilter("Row No.", '>1'); // skip header row
        if not ExcelBuf.FindLast() then
            Error('The uploaded file contains no data rows.');
        LastRow := ExcelBuf."Row No.";

        // Parse each data row (row 1 = header, data starts at row 2)
        for RowNo := 2 to LastRow do begin
            Buffer.Init();
            Buffer."Upload Batch No." := BatchNo;
            Buffer."Row No."          := RowNo - 1; // 1-based data row
            Buffer."Customer No."     := GetCellText(ExcelBuf, RowNo, 1);
            Buffer."Posting Date"     := GetCellDate(ExcelBuf, RowNo, 2);
            Buffer."Document Date"    := GetCellDate(ExcelBuf, RowNo, 3);
            Buffer."External Doc. No." := GetCellText(ExcelBuf, RowNo, 4);
            Buffer."Item No."         := GetCellText(ExcelBuf, RowNo, 5);
            Buffer.Description        := GetCellText(ExcelBuf, RowNo, 6);
            Buffer.Quantity           := GetCellDecimal(ExcelBuf, RowNo, 7);
            Buffer."Unit Price"       := GetCellDecimal(ExcelBuf, RowNo, 8);
            Buffer."Line Discount %"  := GetCellDecimal(ExcelBuf, RowNo, 9);
            Buffer."Dimension 1 Code" := GetCellText(ExcelBuf, RowNo, 10);
            Buffer."Dimension 2 Code" := GetCellText(ExcelBuf, RowNo, 11);
            Buffer.Status             := Buffer.Status::Pending;
            Buffer."Created At"       := CurrentDateTime();
            Buffer.Insert();
            RowCount += 1;
        end;

        exit(RowCount);
    end;

    local procedure GetCellText(var ExcelBuf: Record "Excel Buffer" temporary;
                                Row: Integer; Col: Integer): Text
    begin
        if ExcelBuf.Get(Row, Col) then
            exit(ExcelBuf."Cell Value as Text");
        exit('');
    end;

    local procedure GetCellDate(var ExcelBuf: Record "Excel Buffer" temporary;
                                Row: Integer; Col: Integer): Date
    var
        DateText: Text;
        ParsedDate: Date;
    begin
        DateText := GetCellText(ExcelBuf, Row, Col);
        if DateText = '' then exit(0D);
        if Evaluate(ParsedDate, DateText) then
            exit(ParsedDate);
        exit(0D);
    end;

    local procedure GetCellDecimal(var ExcelBuf: Record "Excel Buffer" temporary;
                                   Row: Integer; Col: Integer): Decimal
    var
        Val: Decimal;
    begin
        if Evaluate(Val, GetCellText(ExcelBuf, Row, Col)) then
            exit(Val);
        exit(0);
    end;
}

Row-Level Validation

After staging, the validator sweeps every row and marks it Valid or Invalid. Validation runs in a single pass with a structured error accumulator — multiple errors per row are captured and concatenated so users fix everything in one go, not one error at a time. 

codeunit 50201 "Sales Invoice Upload Validator"
{
    procedure ValidateBatch(BatchNo: Code[20]): Boolean
    var
        Buffer:   Record "Sales Invoice Upload Buffer";
        HasError: Boolean;
    begin
        Buffer.SetRange("Upload Batch No.", BatchNo);
        Buffer.SetRange(Status, Buffer.Status::Pending);

        if not Buffer.FindSet(true) then
            exit(true);

        repeat
            ValidateRow(Buffer);
            if Buffer.Status = Buffer.Status::Invalid then
                HasError := true;
        until Buffer.Next() = 0;

        exit(not HasError);
    end;

    local procedure ValidateRow(var Buffer: Record "Sales Invoice Upload Buffer")
    var
        Customer: Record Customer;
        Item:     Record Item;
        DimValue: Record "Dimension Value";
        DimSetup: Record "General Ledger Setup";
        Errors:   List of [Text];
        ErrText:  Text;
    begin
        // Customer validation
        if Buffer."Customer No." = '' then
            Errors.Add('Customer No. is mandatory.')
        else if not Customer.Get(Buffer."Customer No.") then
            Errors.Add(StrSubstNo('Customer "%1" does not exist.', Buffer."Customer No."));

        // Posting Date
        if Buffer."Posting Date" = 0D then
            Errors.Add('Posting Date is missing or invalid.');

        // Document Date
        if Buffer."Document Date" = 0D then
            Errors.Add('Document Date is missing or invalid.');

        // External Doc. No.
        if Buffer."External Doc. No." = '' then
            Errors.Add('External Document No. is mandatory.');

        // Item validation
        if Buffer."Item No." = '' then
            Errors.Add('Item No. is mandatory.')
        else if not Item.Get(Buffer."Item No.") then
            Errors.Add(StrSubstNo('Item "%1" does not exist.', Buffer."Item No."))
        else if Item.Blocked then
            Errors.Add(StrSubstNo('Item "%1" is blocked.', Buffer."Item No."));

        // Quantity
        if Buffer.Quantity <= 0 then
            Errors.Add('Quantity must be greater than zero.');

        // Unit Price
        if Buffer."Unit Price" < 0 then
            Errors.Add('Unit Price cannot be negative.');

        // Line Discount %
        if (Buffer."Line Discount %" < 0) or (Buffer."Line Discount %" > 100) then
            Errors.Add('Line Discount % must be between 0 and 100.');

        // Dimension validation
        DimSetup.Get();
        if Buffer."Dimension 1 Code" <> '' then
            if not DimValue.Get(DimSetup."Global Dimension 1 Code", Buffer."Dimension 1 Code") then
                Errors.Add(StrSubstNo('Dimension 1 value "%1" is invalid.', Buffer."Dimension 1 Code"));

        // Set status
        if Errors.Count = 0 then begin
            Buffer.Status := Buffer.Status::Valid;
            Buffer."Error Message" := '';
        end else begin
            Buffer.Status := Buffer.Status::Invalid;
            foreach ErrText in Errors do
                Buffer."Error Message" += '• ' + ErrText + '  ';
        end;

        Buffer."Processed At" := CurrentDateTime();
        Buffer.Modify();
    end;
}

Validation Rules Summary

Customer No. exists and is not blocked
Posting Date and Document Date are valid, non-zero dates
External Document No. is present and unique per customer
Item No. exists and is not blocked
Quantity is greater than zero
Unit Price is non-negative
Line Discount % is between 0 and 100
Dimension values exist in the dimension table

Error Report Generation

After validation, generate a downloadable error report. This gives the finance team a single file to work from — every invalid row, every error message, ready to fix and re-upload. 

codeunit 50202 "Sales Invoice Upload Reporter"
{
    procedure ExportErrorReport(BatchNo: Code[20])
    var
        Buffer:   Record "Sales Invoice Upload Buffer";
        ExcelBuf: Record "Excel Buffer" temporary;
        RowNo:    Integer;
        OutStream: OutStream;
        FileName: Text;
    begin
        Buffer.SetRange("Upload Batch No.", BatchNo);
        Buffer.SetRange(Status, Buffer.Status::Invalid);

        if not Buffer.FindSet() then begin
            Message('No validation errors found. All rows are valid.');
            exit;
        end;

        // Write header row
        RowNo := 1;
        AddCell(ExcelBuf, RowNo, 1, 'Row No.',       true);
        AddCell(ExcelBuf, RowNo, 2, 'Customer No.',   true);
        AddCell(ExcelBuf, RowNo, 3, 'External Doc.',  true);
        AddCell(ExcelBuf, RowNo, 4, 'Item No.',       true);
        AddCell(ExcelBuf, RowNo, 5, 'Error(s)',       true);

        // Write error rows
        repeat
            RowNo += 1;
            AddCell(ExcelBuf, RowNo, 1, Format(Buffer."Row No."),         false);
            AddCell(ExcelBuf, RowNo, 2, Buffer."Customer No.",            false);
            AddCell(ExcelBuf, RowNo, 3, Buffer."External Doc. No.",       false);
            AddCell(ExcelBuf, RowNo, 4, Buffer."Item No.",                false);
            AddCell(ExcelBuf, RowNo, 5, Buffer."Error Message",           false);
        until Buffer.Next() = 0;

        // Download the report
        FileName := 'Upload_Errors_' + BatchNo + '.xlsx';
        ExcelBuf.CreateNewBook('Errors');
        ExcelBuf.WriteSheet('Validation Errors', CompanyName(), UserId());
        ExcelBuf.CloseBook();
        ExcelBuf.SetFriendlyFilename(FileName);
        ExcelBuf.OpenExcel();
    end;

    local procedure AddCell(var ExcelBuf: Record "Excel Buffer" temporary;
                            Row: Integer; Col: Integer; Value: Text; Bold: Boolean)
    begin
        ExcelBuf.Init();
        ExcelBuf.Validate("Row No.", Row);
        ExcelBuf.Validate("Column No.", Col);
        ExcelBuf."Cell Value as Text" := Value;
        ExcelBuf.Bold := Bold;
        ExcelBuf.Insert();
    end;
}

Background Posting via Job Queue

Once all rows are marked Valid, posting is handed off to a Job Queue Entry. This frees the user's browser session immediately and ensures posting runs in a clean background context — important for large batches where transaction timeouts are a risk. 

codeunit 50203 "Sales Invoice Upload Poster"
{
    TableNo = "Job Queue Entry";

    trigger OnRun()
    var
        Buffer:  Record "Sales Invoice Upload Buffer";
        BatchNo: Code[20];
    begin
        BatchNo := Rec."Parameter String";

        Buffer.SetRange("Upload Batch No.", BatchNo);
        Buffer.SetRange(Status, Buffer.Status::Valid);

        if not Buffer.FindSet(true) then
            exit;

        repeat
            Commit(); // each invoice in its own transaction
            if not TryPostInvoice(Buffer) then begin
                Buffer.Status        := Buffer.Status::Failed;
                Buffer."Error Message" := GetLastErrorText();
                Buffer."Processed At"  := CurrentDateTime();
                Buffer.Modify();
            end;
        until Buffer.Next() = 0;
    end;

    [TryFunction]
    local procedure TryPostInvoice(var Buffer: Record "Sales Invoice Upload Buffer")
    var
        SalesHeader: Record "Sales Header";
        SalesLine:   Record "Sales Line";
        Item:        Record Item;
        PostedNo:    Code[20];
    begin
        // Create Sales Invoice Header
        SalesHeader.Init();
        SalesHeader."Document Type" := SalesHeader."Document Type"::Invoice;
        SalesHeader.Insert(true);
        SalesHeader.Validate("Sell-to Customer No.", Buffer."Customer No.");
        SalesHeader.Validate("Posting Date",         Buffer."Posting Date");
        SalesHeader.Validate("Document Date",        Buffer."Document Date");
        SalesHeader."External Document No."        := Buffer."External Doc. No.";

        if Buffer."Dimension 1 Code" <> '' then
            SalesHeader.Validate("Shortcut Dimension 1 Code", Buffer."Dimension 1 Code");
        if Buffer."Dimension 2 Code" <> '' then
            SalesHeader.Validate("Shortcut Dimension 2 Code", Buffer."Dimension 2 Code");
        SalesHeader.Modify(true);

        // Create Sales Line
        SalesLine.Init();
        SalesLine."Document Type" := SalesHeader."Document Type";
        SalesLine."Document No."  := SalesHeader."No.";
        SalesLine."Line No."      := 10000;
        SalesLine.Insert(true);
        SalesLine.Validate(Type, SalesLine.Type::Item);
        SalesLine.Validate("No.", Buffer."Item No.");

        if Buffer.Description <> '' then
            SalesLine.Description := Buffer.Description;

        SalesLine.Validate(Quantity,          Buffer.Quantity);
        SalesLine.Validate("Unit Price",      Buffer."Unit Price");
        SalesLine.Validate("Line Discount %", Buffer."Line Discount %");
        SalesLine.Modify(true);

        // Post
        PostedNo := CODEUNIT.Run(CODEUNIT::"Sales-Post", SalesHeader) ?
                    SalesHeader."Last Posting No." : '';

        // Update buffer
        Buffer.Status          := Buffer.Status::Posted;
        Buffer."BC Invoice No." := PostedNo;
        Buffer."Processed At"  := CurrentDateTime();
        Buffer.Modify();
    end;
}

Enqueueing the Job

procedure EnqueuePosting(BatchNo: Code[20])
var
    JobQueueEntry: Record "Job Queue Entry";
begin
    JobQueueEntry.Init();
    JobQueueEntry."Object Type to Run"     := JobQueueEntry."Object Type to Run"::Codeunit;
    JobQueueEntry."Object ID to Run"       := Codeunit::"Sales Invoice Upload Poster";
    JobQueueEntry."Parameter String"       := BatchNo;
    JobQueueEntry.Description             := 'Sales Invoice Upload – Batch ' + BatchNo;
    JobQueueEntry."Maximum No. of Attempts to Run" := 1;
    JobQueueEntry.Insert(true);
    CODEUNIT.Run(CODEUNIT::"Job Queue - Enqueue", JobQueueEntry);
    Message('Posting started in background. Batch: %1', BatchNo);
end;

Tips & Edge Cases

01

Duplicate Detection

Before posting, check if a Sales Invoice with the same External Doc. No. and Customer already exists. Block the row with a clear message rather than creating a duplicate.

02

Multi-Line Invoices

Group rows by External Doc. No. per customer. All rows sharing the same External Doc. No. become lines on a single invoice header — don't create a new header per row.

03

Batch Cleanup

Add a scheduled cleanup codeunit that deletes staging records older than 30 days. Staging tables grow fast — don't let them become technical debt.

04

Partial Reprocessing

After fixing errors, allow users to reprocess only the Invalid rows of a batch rather than re-uploading the whole file. Filter the batch by status before validation.
