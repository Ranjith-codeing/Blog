SharePoint Integration with Dynamics 365 Business Central

Integrating SharePoint with Business Central enables organizations to manage documents efficiently while keeping transactional data clean inside ERP.

🔷 Business Use Case

Users upload documents such as invoices or purchase orders to SharePoint, and Business Central links those files to relevant records.

  • Vendor Invoice attachments
  • Sales Order documents
  • Customer supporting files

🔷 Integration Approaches

  • REST API (Recommended) – Full control via AL
  • Power Automate – Event-based integration
  • Azure Functions – Middleware for complex scenarios

🔷 Architecture Flow

User → Upload File → SharePoint → API → Business Central → Store Link

🔷 Key Components in Business Central

Document Mapping Table

Field Description
Document ID Unique identifier
Table ID BC table reference
Record No. Document number
SharePoint URL File location

🔷 Authentication

Use OAuth 2.0 via Azure AD. Store:

  • Client ID
  • Client Secret
  • Tenant ID

🔷 Sample AL Code

procedure UploadToSharePoint(FileName: Text; FileContent: InStream): Text var HttpClient: HttpClient; HttpContent: HttpContent; Response: HttpResponseMessage; begin HttpContent.WriteFrom(FileContent); if not HttpClient.Put('SHAREPOINT_API_URL', HttpContent, Response) then Error('Upload failed'); exit('File Uploaded Successfully'); end;

🔷 Validation Logic

  • File name mandatory
  • Allowed file types (PDF, DOCX)
  • Record existence validation

🔷 Error Handling

  • Log API failures
  • Retry mechanism
  • User-friendly messages

🔷 Performance Considerations

🔷 Benefits

🔷 Conclusion

SharePoint integration provides a scalable and secure document management solution for Business Central.
