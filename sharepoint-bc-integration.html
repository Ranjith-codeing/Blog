SharePoint Integration with Business Central

Designing a scalable, secure document management layer using Microsoft SharePoint — with full architecture walkthrough, AL code, and real-world patterns.

Ranjith · March 2026

Every Dynamics 365 Business Central project eventually faces the same question: where do documents live? Storing files directly in the ERP database bloats it, slows backups, and complicates audits. SharePoint — already part of the Microsoft 365 ecosystem — is the natural answer. This guide walks through a production-ready integration from first principles.

Why SharePoint?

Business Central supports native attachment storage via the Tenant Media and Document Attachment tables, but this comes with limitations at scale. When attachment sizes climb into the gigabytes and record counts stretch into the hundreds of thousands, database performance degrades noticeably — and cloud-hosted environments have storage caps that add up quickly.

SharePoint Online, as a Microsoft-first platform, offers a compelling alternative:

  • Unlimited document storage relative to ERP database cost
  • Native Microsoft 365 integration — same identity, permissions, and compliance stack
  • Versioning and audit trails built into the platform
  • Familiar UI for end users who already use Teams and OneDrive
  • Governance features like retention policies, sensitivity labels, and DLP
💡
Key insight: SharePoint should be treated as a document store, not a collaboration hub, in this pattern. Business Central remains the system of record; SharePoint is the file cabinet.

Solution Architecture

The integration follows a three-layer pattern. Business Central handles transactional logic and stores only a reference (the SharePoint URL) — never the file bytes themselves.

🏢
Business Central
Document links, metadata, validation logic
→
⚙️
Azure AD OAuth
Token acquisition, app registration, scopes
→
📁
SharePoint Online
File storage, versioning, permissions

Data Model

Two custom tables anchor the integration:

Table Purpose Key Fields
SP Document Mapping Links BC records to SharePoint files Record Type, Record No., SharePoint URL, File Name, Uploaded At
SP Integration Setup Holds connection configuration Tenant ID, Client ID, Client Secret, Site URL, Library Name

Prerequisites & Setup

Before writing a single line of AL, the Azure side must be configured correctly. Skipping this step is the most common source of integration failures.

Azure App Registration

  1. Go to Azure Active Directory → App Registrations → New Registration
  2. Name it something descriptive: BC-SharePoint-Integration
  3. Set the redirect URI to your BC environment URL (for delegated flows) or leave blank (for client credentials)
  4. Under API Permissions, add:
    • Sites.ReadWrite.All (Application permission)
    • Files.ReadWrite.All (Application permission)
  5. Click Grant admin consent — this is mandatory for app-only flows
  6. Under Certificates & Secrets, create a new client secret and copy it immediately
⚠️
Security note: Store the Client Secret in Business Central's Isolated Storage, never in a plain text field. Rotate secrets every 12 months and track expiry dates proactively.

SharePoint Site Structure

Organise your SharePoint library to mirror your BC document types:

/sites/BusinessCentral/

AL Code Walkthrough

Below is the core codeunit that manages the token acquisition and file upload. This is production-ready AL, not pseudocode.

Codeunit: SP Integration Manager

codeunit 50100 "SP Integration Manager"
{
    procedure UploadDocumentToSharePoint(
        RecordType: Text;
        RecordNo: Code[20];
        FileName: Text;
        FileContent: InStream
    ): Text
    var
        Setup: Record "SP Integration Setup";
        Mapping: Record "SP Document Mapping";
        AccessToken: Text;
        UploadUrl: Text;
        SharePointUrl: Text;
    begin
        // Fetch setup
        Setup.Get();
        Setup.TestField("Site URL");
        Setup.TestField("Library Name");

        // Acquire OAuth token
        AccessToken := AcquireAccessToken(Setup);

        // Build upload URL for SharePoint REST API
        UploadUrl := Setup."Site URL" +
            '/_api/web/GetFolderByServerRelativeUrl(''' +
            Setup."Library Name" + '/' + RecordType + '/' + RecordNo +
            ''')/Files/add(url=''' + FileName +
            ''',overwrite=true)';

        // Upload the file
        SharePointUrl := UploadFileToSharePoint(
            UploadUrl, AccessToken, FileContent, FileName);

        // Store the mapping
        Mapping.Init();
        Mapping."Record Type"  := RecordType;
        Mapping."Record No."   := RecordNo;
        Mapping."File Name"    := FileName;
        Mapping."SharePoint URL" := SharePointUrl;
        Mapping."Uploaded At"  := CurrentDateTime();
        Mapping."Uploaded By"  := UserId();
        Mapping.Insert(true);

        exit(SharePointUrl);
    end;

    local procedure AcquireAccessToken(Setup: Record "SP Integration Setup"): Text
    var
        Client: HttpClient;
        Request: HttpRequestMessage;
        Response: HttpResponseMessage;
        Content: HttpContent;
        Headers: HttpHeaders;
        JsonObj: JsonObject;
        TokenText: Text;
        Body: Text;
        ClientSecret: Text;
    begin
        // Retrieve secret from Isolated Storage
        if not IsolatedStorage.Get('SP_ClientSecret', DataScope::Company, ClientSecret) then
            Error('SharePoint client secret not configured.');

        Body := 'grant_type=client_credentials' +
                '&client_id=' + Setup."Client ID" +
                '&client_secret=' + ClientSecret +
                '&scope=https%3A%2F%2Fgraph.microsoft.com%2F.default';

        Content.WriteFrom(Body);
        Content.GetHeaders(Headers);
        Headers.Remove('Content-Type');
        Headers.Add('Content-Type', 'application/x-www-form-urlencoded');

        Request.Method := 'POST';
        Request.SetRequestUri(
            'https://login.microsoftonline.com/' + Setup."Tenant ID" + '/oauth2/v2.0/token');
        Request.Content := Content;

        Client.Send(Request, Response);

        if not Response.IsSuccessStatusCode() then
            Error('Token acquisition failed. Status: %1', Response.HttpStatusCode());

        Response.Content.ReadAs(TokenText);
        JsonObj.ReadFrom(TokenText);
        exit(GetJsonValue(JsonObj, 'access_token'));
    end;
}

Codeunit: SP Document Page Extension

This page extension wires the upload action into Vendor Ledger Entries — the most common use case. 

pageextension 50100 "Vendor Ledger SP Ext" extends "Vendor Ledger Entries"
{
    actions
    {
        addlast(processing)
        {
            action(UploadToSharePoint)
            {
                Caption = 'Upload to SharePoint';
                Image = Attach;
                ApplicationArea = All;
                ToolTip = 'Upload a document and link it to this entry.';

                trigger OnAction()
                var
                    SPMgr: Codeunit "SP Integration Manager";
                    InStr: InStream;
                    FileName: Text;
                begin
                    if not UploadIntoStream('Select Document', '', '', FileName, InStr) then
                        exit;

                    SPMgr.UploadDocumentToSharePoint(
                        'VendorInvoices',
                        Format(Rec."Entry No."),
                        FileName,
                        InStr);

                    Message('Document uploaded and linked successfully.');
                end;
            }
        }
    }
}
💡
The UploadIntoStream built-in handles browser file picker dialogs in both web client and mobile — no custom dialogs needed.

Validation & Error Handling

A production integration must fail gracefully. Uploads can fail for dozens of reasons — network blips, SharePoint throttling, file type restrictions, or expired tokens. Each failure mode needs a distinct path.

Upload Log Table

Track every attempt — success or failure — so operations teams can diagnose issues without digging through event logs:

Field Type Purpose
Entry No. Integer Auto-key
Record Type Text[50] Source document type
Record No. Code[20] Source document number
File Name Text[250] Original file name
Status Enum Pending / Success / Failed
Error Message Text[2048] Full error from SharePoint API
Retry Count Integer Number of retry attempts
Created At DateTime First attempt timestamp
Processed At DateTime Last attempt timestamp

Retry Job Queue

Failed uploads are picked up by a Job Queue Entry running every 15 minutes. The retry codeunit queries for Status = Failed and Retry Count < 3, then re-attempts the upload:

codeunit 50101 "SP Retry Failed Uploads"
{
    TableNo = "Job Queue Entry";

    trigger OnRun()
    var
        Log: Record "SP Upload Log";
        SPMgr: Codeunit "SP Integration Manager";
    begin
        Log.SetRange(Status, Log.Status::Failed);
        Log.SetFilter("Retry Count", '<%1', 3);

        if not Log.FindSet(true) then
            exit;

        repeat
            Commit(); // isolate each retry in its own transaction
            if not TryUploadFromLog(Log, SPMgr) then begin
                Log."Error Message" := GetLastErrorText();
                Log."Retry Count"   += 1;
                Log."Processed At"  := CurrentDateTime();
                Log.Modify();
            end;
        until Log.Next() = 0;
    end;

Validation Rules

File extension must be in the allowed list (PDF, DOCX, XLSX, PNG, JPG)
File size must not exceed 50 MB per upload
Duplicate file name detection per record (warn, not block)
Record must exist in BC before allowing attachment
SharePoint setup must be configured before any upload action is enabled

Performance & Scalability

At low volumes, any approach works. The design choices that matter show up when you're processing hundreds of attachments per day across thousands of BC records.

Key Optimisation Techniques

01

Background Upload via Job Queue

Never upload synchronously in a page action if the file is larger than ~2 MB. Enqueue the upload as a Job Queue Entry and give the user immediate feedback.

02

Token Caching

OAuth tokens are valid for 3600 seconds. Cache the token in Isolated Storage with an expiry check — avoid a round-trip to Azure AD on every upload.

03

Chunked Upload for Large Files

SharePoint's REST API supports chunked uploads via StartUpload / ContinueUpload / FinishUpload. Use this for files above 10 MB.

04

Folder Pre-creation

Create the record-level folder in SharePoint at document creation time in BC, not at upload time. Eliminates a REST call per upload.

SharePoint API Rate Limits

SharePoint Online enforces throttling. At scale, implement exponential back-off:

// Simplified back-off logic
MaxRetries := 3;
DelayMs := 500; // start at 500ms

for Attempt := 1 to MaxRetries do begin
    if TryCallSharePointApi(...) then
        exit; // success

    if Response.HttpStatusCode() = 429 then begin
        Sleep(DelayMs);
        DelayMs *= 2; // 500 → 1000 → 2000ms
    end else
        Error('SharePoint error: %1', Response.HttpStatusCode());
end;

Security Considerations

Document management is a compliance surface. Get these right from day one — retrofitting security is painful.

Secrets Management

  • Store Client Secret in Isolated Storage scoped to DataScope::Company
  • Never expose secrets on a setup page — write-only fields with masked display
  • Implement a Test Connection action so admins can validate without seeing the secret

SharePoint Permissions

  • Use a dedicated service account (app registration) with minimum required permissions
  • Grant Sites.Selected instead of Sites.ReadWrite.All if targeting a specific site only — available since 2021
  • Apply SharePoint site column Sensitivity Labels for documents that contain PII

Business Central Permission Sets

Permission Set Users Access Level
SP-ADMIN IT / ERP Admins Full setup + read logs
SP-UPLOAD AP / AR Teams Upload documents, view links
SP-READ Auditors View document links only
🔒
Enable SharePoint audit logging and send those logs to a Log Analytics workspace. This gives you a searchable, tamper-proof audit trail that satisfies most compliance requirements (ISO 27001, SOC 2, GDPR).

Conclusion

A well-implemented SharePoint integration transforms how Business Central handles documents. You eliminate database bloat, gain a proper compliance layer, and give end users access to files from the tools they already use — all without rebuilding the wheel.

The pattern described here — reference-based linking, background uploads, retry queues, and scoped permissions — is a reliable production template. Adapt the folder structure and permission sets to your organisation's conventions, and you have a foundation that scales from a 10-user pilot to an enterprise-wide rollout.
